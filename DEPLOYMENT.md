## Варианты хостинга

### 1. Бесплатные хостинги (для начала)

#### GitHub Pages (рекомендуется для статических сайтов)
1. Создайте аккаунт на [GitHub.com](https://github.com)
2. Создайте новый репозиторий (например, `lockwood-trading`)
3. Загрузите все файлы в репозиторий
4. В настройках репозитория включите GitHub Pages
5. Сайт будет доступен по адресу: `https://ваш-username.github.io/lockwood-trading`

##### Настройка кастомного домена на GitHub Pages

**Да, вы можете использовать свой домен!** GitHub Pages полностью поддерживает кастомные домены.

**Шаги настройки:**

1. **В настройках GitHub репозитория:**
   - Перейдите в `Settings` > `Pages`
   - В разделе `Custom domain` введите ваш домен (например: `lockwoodtrading.com` или `www.lockwoodtrading.com`)
   - Нажмите `Save`
   - GitHub автоматически создаст файл `CNAME` в репозитории

2. **Настройка DNS у регистратора домена:**

   **Вариант A: Использовать поддомен (www)**
   - Создайте **CNAME запись**:
     - Имя: `www`
     - Значение: `ваш-username.github.io`
     - TTL: 3600 (или по умолчанию)

   **Вариант B: Использовать корневой домен (без www)**
   - Создайте **A записи** (нужно 4 записи):
     ```
     Имя: @ (или пусто)
     Значение: 185.199.108.153
     TTL: 3600
     
     Имя: @
     Значение: 185.199.109.153
     TTL: 3600
     
     Имя: @
     Значение: 185.199.110.153
     TTL: 3600
     
     Имя: @
     Значение: 185.199.111.153
     TTL: 3600
     ```
   - Или создайте **ALIAS/ANAME запись** (если поддерживается):
     - Имя: `@`
     - Значение: `ваш-username.github.io`

   **Вариант C: И www, и без www (рекомендуется)**
   - Настройте оба варианта:
     - CNAME для `www` → `ваш-username.github.io`
     - A записи для `@` → IP адреса GitHub (см. выше)
   - В настройках GitHub укажите домен с `www` (например: `www.lockwoodtrading.com`)
   - GitHub автоматически перенаправит `lockwoodtrading.com` на `www.lockwoodtrading.com`

3. **Включите HTTPS (рекомендуется):**
   - В настройках GitHub Pages включите опцию `Enforce HTTPS`
   - GitHub автоматически выдаст SSL сертификат через Let's Encrypt
   - Подождите несколько минут после настройки DNS

4. **Проверка:**
   - Подождите 10-60 минут для распространения DNS изменений
   - Проверьте на [whatsmydns.net](https://www.whatsmydns.net)
   - Откройте ваш домен в браузере

**Важные моменты:**
- DNS изменения могут занять до 48 часов (обычно 10-60 минут)
- Убедитесь, что домен правильно указан в настройках GitHub (с www или без)
- GitHub автоматически создаст файл `CNAME` в корне репозитория - не удаляйте его
- Если используете корневой домен (@), GitHub рекомендует также настроить www версию

**Пример настройки для домена `lockwoodtrading.com`:**

В панели управления DNS (у регистратора домена):
```
Тип    Имя    Значение                    TTL
A      @      185.199.108.153             3600
A      @      185.199.109.153             3600
A      @      185.199.110.153             3600
A      @      185.199.111.153             3600
CNAME  www    ваш-username.github.io      3600
```

В GitHub: укажите `www.lockwoodtrading.com` в Custom domain

## Способ 1: Загрузка через FTP/SFTP

### Шаги:
1. **Получите данные для FTP** от вашего хостинг-провайдера:
   - FTP сервер (например: `ftp.yourdomain.com`)
   - Логин (username)
   - Пароль (password)
   - Порт (обычно 21 для FTP или 22 для SFTP)

2. **Используйте FTP клиент**:
   - **FileZilla** (бесплатный) - [filezilla-project.org](https://filezilla-project.org)
   - **Cyberduck** (Mac/Windows) - [cyberduck.io](https://cyberduck.io)
   - **WinSCP** (Windows) - [winscp.net](https://winscp.net)

3. **Подключитесь к серверу**:
   - Введите данные FTP
   - Подключитесь

4. **Загрузите файлы**:
   - Найдите папку `public_html` или `www` или `htdocs` (зависит от хостинга)
   - Загрузите ВСЕ файлы из папки проекта в эту директорию
   - Убедитесь, что `index.html` находится в корневой папке

## Способ 2: Загрузка через панель управления (cPanel)

1. Войдите в **cPanel** вашего хостинга
2. Найдите **File Manager**
3. Перейдите в папку `public_html`
4. Нажмите **Upload** и загрузите все файлы
5. Или используйте **Extract** если загружаете ZIP архив

## Способ 3: Через командную строку (SSH)

Если у вас есть доступ по SSH:

```bash
# Перейдите в папку проекта
cd /Users/dmitrijudenkov/Desktop/LOCKWOOD

# Создайте архив (опционально)
zip -r lockwood-site.zip . -x "*.md" "HERO_IMAGE_INSTRUCTIONS.md"

# Загрузите через SCP
scp -r * username@your-server.com:/path/to/public_html/

# Или используйте rsync
rsync -avz --exclude='*.md' --exclude='HERO_IMAGE_INSTRUCTIONS.md' ./ username@your-server.com:/path/to/public_html/
```

### Проверка после загрузки:
1. Откройте сайт в браузере
2. Проверьте все страницы: главная, Staff, Shipping
3. Убедитесь, что изображения загружаются
4. Проверьте работу форм и навигации
5. Проверьте на мобильных устройствах

### Настройка домена:
- Если используете свой домен, настройте DNS записи:
  - A запись указывает на IP сервера

- Убедитесь, что видео `hero-background.mp4` не слишком большое (рекомендуется < 5MB)
- Можно использовать сжатие изображений перед загрузкой
- Включите GZIP сжатие на сервере (обычно включено по умолчанию)

## Быстрый старт с GitHub Pages

Если хотите быстро запустить сайт:

```bash
# 1. Инициализируйте git репозиторий
cd /Users/dmitrijudenkov/Desktop/LOCKWOOD
git init

# 2. Добавьте все файлы
git add .

# 3. Сделайте первый коммит
git commit -m "Initial commit"

# 4. Создайте репозиторий на GitHub и подключите
git remote add origin https://github.com/ваш-username/lockwood-trading.git
git branch -M main
git push -u origin main

# 5. В настройках GitHub репозитория:
# Settings > Pages > Source: main branch > Save
```

Если возникнут проблемы:

- Убедитесь, что `index.html` находится в правильной директории
- Проверьте логи ошибок в панели управления хостингом
